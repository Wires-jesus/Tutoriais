Pré-requisitos

  • sqlplus.exe disponível no PATH (Oracle Database Home instalado)
  • GitHub Copilot CLI ativo no VS Code
  • Arquivo db.env preenchido com os perfis desejados
1

Arquivo de credenciais com múltiplos perfis

O arquivo fica em um diretório fora do repositório Git:

C:\Analises-Copilot\Conexao_banco\db.env

Cada perfil usa um prefixo em maiúsculas com o nome do owner:

# Perfil: local
LOCAL_USER=seu_usuario
LOCAL_PASS=sua_senha
LOCAL_HOST=127.0.0.1
LOCAL_PORT=1521
LOCAL_SERVICE=orcl

# Perfil: monitorpdvmiddle
MONITORPDVMIDDLE_USER=seu_usuario
MONITORPDVMIDDLE_PASS=sua_senha
MONITORPDVMIDDLE_HOST=host_do_banco
MONITORPDVMIDDLE_PORT=1521
MONITORPDVMIDDLE_SERVICE=nome_do_servico
⚠️ Nunca adicione o arquivo db.env ao repositório Git. Substitua os valores pelos dados reais do seu ambiente.
2

Perfis disponíveis

🗄️ local
HOST: 127.0.0.1:1521/orcl USER: local
"Conecte no owner local"
🗄️ monitorpdvmiddle
HOST: host_do_banco:1521/service USER: seu_usuario
"Conecte no owner monitorpdvmiddle"
ℹ️ Para adicionar um novo perfil, inclua um novo bloco no db.env seguindo o padrão NOVOOWNER_USER, NOVOOWNER_PASS, etc.
3

Uso diário

Comando ao CopilotPerfil utilizado
"Conecte no owner local"LOCAL_*
"Conecte no owner monitorpdvmiddle"MONITORPDVMIDDLE_*
"Conecte no owner novoowner"NOVOOWNER_*

Exemplos de análises após conexão:

"Quais os campos da tabela PCDESCONTOFIDELIDADE?"
→ Descreve colunas via ALL_TAB_COLUMNS
"Liste as tabelas que referenciam PCPRODUT"
→ Consulta ALL_DEPENDENCIES
"Quantos registros tem PCGRUPOFIDELIDADE?"
→ Executa SELECT COUNT(*)
"Descreva a estrutura da tabela PCCLIENT"
→ Retorna DDL completo

💡

Capacidades com Copilot conectado

🤖
Com o Copilot conectado ao banco, é possível realizar análises em tempo real:
  • Estrutura de tabelas (ALL_TAB_COLUMNS)
  • Dependências entre objetos (ALL_DEPENDENCIES)
  • Comparação entre estrutura do banco e objetos do repositório
  • Validação de dados e contagens em tempo real

O Copilot não acessa o banco automaticamente — a conexão só ocorre quando solicitada com o comando "conecte no owner <nome>".

🔌 Conexão Oracle via SQLPlus + Copilot CLI

Fluxo de conexão diária para análises assistidas pelo GitHub Copilot
📅 Julho de 2026 🔖 v1.0 📂 Base de Conhecimento
💡

As credenciais ficam em um arquivo local fora do repositório Git. Quando o analista solicita "Conecte-se ao banco", o Copilot CLI lê o arquivo, abre conexão via sqlplus e executa consultas sob demanda — sem expor credenciais em commits.

🤖

⚡
sqlplus
Conexão Oracle

📊
Análise
Resultados no terminal
✓

Pré-requisitos

  • sqlplus.exe disponível no PATH (Oracle Database Home instalado)
  • GitHub Copilot CLI ativo no VS Code
  • Credenciais de acesso ao banco (usuário, senha, host, porta, service name)
1

Criar o arquivo de credenciais

Crie (ou edite) o arquivo no seguinte caminho — fora do repositório Git:

C:\Analises-Copilot\Conexao_banco\db.env

Conteúdo do arquivo:

ORACLE_USER=seu_usuario
ORACLE_PASS=sua_senha
ORACLE_HOST=host_do_banco
ORACLE_PORT=1521
ORACLE_SERVICE=nome_do_servico
⚠️ Substitua os valores pelos dados reais do seu ambiente. Nunca adicione este arquivo ao repositório Git.
2

Uso diário

Com o GitHub Copilot CLI aberto no terminal do VS Code, solicite:

"Conecte-se ao banco"

O Copilot irá ler o db.env, conectar via sqlplus e ficará pronto para consultas.

Exemplos de análises após conexão:

"Quais os campos da tabela PCDESCONTOFIDELIDADE?"
→ Descreve colunas, tipos e nullability
"Liste as tabelas que referenciam PCPRODUT"
→ Consulta ALL_DEPENDENCIES
"Descreva a estrutura da tabela PCCLIENT"
→ Retorna DDL via ALL_TAB_COLUMNS
"Quantos registros tem PCDESCONTOFIDELIDADE?"
→ Executa SELECT COUNT(*)
